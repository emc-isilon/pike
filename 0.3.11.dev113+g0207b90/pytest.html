Pytest Integration¶

Pike provides pytest marks and fixtures to assist in writing SMB2 protocol tests using the pytest framework.

Typically, tests using these fixtures will leverage the PIKE_* environment variables to determine where to connect and other protocol features.

Pike/pytest integration

Available markers:

  • require_dialect(min_dialect, max_dialect): raise a DialectMissing exception if the target server does not support a dialect in the given range. This exception will be raised after Negotiate.

  • require_capability(global_capabilities): raise a CapabilityMissing exception if the target server does not support all of the OR’d global capability flags. This exception will be raised after Negotiate.

  • require_share_capability(share_capabilities): raise a ShareCapabilityMissing exception if the target share does not support all of the OR’d share capability flags. This exception will be raised after Tree Connect.

pike_TreeConnect¶

pike.pytest_support.pike_TreeConnect(request)[source]¶

Function-scope fixture returns a wrapper function around pike.TreeConnect.

The wrapper-function must be called to establish the connection and the returned object may be used as a contextmanager.

The wrapper-function will pass through any require_dialect, require_capabilities, and require_share_capabilities marks on the test item and raise an exception if the target server doesn’t support the required values.

Returns

Callable[…, pike.TreeConnect]

pike_tmp_path¶

pike.pytest_support.pike_tmp_path(pike_TreeConnect)[source]¶

Function-scope fixture reaturns a PikePath rooted at a timestamped subdirectory of the share.

The temporary directory is not deleted during tearDown and should be removed by the test case itself.

A function’s pike_TreeConnect() fixture will access the same share, however, via a separate SMB2 session.

This fixture is similar to pytest’s built in tmp_path fixture.
